# API 模型配置说明

本项目支持四种 API 提供商，通过右上角设置按钮进入配置面板。

---

## 内置提供商

### Claude（Anthropic）

| 字段 | 值 |
|------|-----|
| API 地址 | `https://api.anthropic.com/v1/messages` |
| 默认模型 | `claude-sonnet-4-6` |
| Key 格式 | `sk-ant-` 开头 |
| 获取地址 | https://console.anthropic.com |

> 注意：Claude 使用独立协议格式，**不经过本地代理**，直接从浏览器发送请求，需开启 CORS 直连权限。

---

### ChatGPT（OpenAI）

| 字段 | 值 |
|------|-----|
| API 地址 | `https://api.openai.com/v1/chat/completions` |
| 默认模型 | `gpt-4o` |
| Key 格式 | `sk-` 开头 |
| 获取地址 | https://platform.openai.com/api-keys |

---

### Kimi（Moonshot）

| 字段 | 值 |
|------|-----|
| API 地址 | `https://api.moonshot.cn/v1/chat/completions` |
| 默认模型 | `moonshot-v1-8k` |
| Key 格式 | `sk-` 开头 |
| 获取地址 | https://platform.moonshot.cn |

---

## 自定义提供商

选择「自定义」后，需手动填写三项：

| 字段 | 要求 |
|------|------|
| API Key | 你的密钥 |
| API 地址 | **完整 endpoint**，必须包含路径，例如 `https://api.xixixixi.cloud/v1/chat/completions` |
| 模型名称 | 服务商指定的模型 ID，例如 `kimi-2.6` |

**常见错误**：API 地址只填了域名（如 `https://api.xixixixi.cloud`），服务器返回 HTML 首页，导致报错：

```
Unexpected token '<', "<!DOCTYPE "... is not valid JSON
```

必须加上 `/v1/chat/completions` 路径。

自定义提供商需兼容 OpenAI 接口格式（即响应体含 `choices[0].message.content` 字段）。

---

## 请求流程

```
浏览器 index.html
    │
    ├── Claude → 直接请求 Anthropic API（CORS 直连）
    │
    └── 其他 → POST /api-proxy（本地代理）
                    │
                    └── curl 转发到目标 API（绕过 CORS 限制）
```

使用 OpenAI 格式的提供商需先启动本地代理：

```bash
python3 proxy.py
# 然后访问 http://localhost:8080
```
